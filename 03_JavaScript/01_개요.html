개요

        JavaScript는 1995년 브라우저에 동작을 넣기 위해 브렌던 아이크에 의해 만들어진 언어
        브라우저 전쟁을 거쳐 ECMAScript라는 표준으로 통일됐고, ES6 이후
        현대적 언어로 재탄생을 거쳐서 지금은 가장 웹에서 많이 영향력이 있는 언어로 자리잡음

스크립트 언어

애플리케이션의 제어를 사용자의 요구에 맞게 수행할 수 있도록 빠르게 배우고 짧은 소스코드로 상호작용한 가능한 언어들

자바스크립트

웹상에서 가장 많이 사용하는 스크립트 언어
현시대 웹에서 화면을 만들어내는 기반기술로 사용되는것이 가장 큰 특징
(Angular, Vue, React) - 컴파일 : 컴파일러가 컴파일 단계를 거쳐서 실행
- 인터프리터 : 필요할 때 마다 한 줄 한 줄 바로바로 실행
JIT 컴파일러 => 자주 실행되는 "핫 코드"를 기계어로 재 컴파일 자바 / 자바스크립트 => 컴파일 + 인터프리터

  • 함수를 일급객체로 취급함
  • 문법이 쉬움
    접근제한자 개념이 X,함수 반환형 X, 변수 타입 지정 X(타입추론)
  • 계속 발전하고 피드백이 빠름 확장성이 좋아 배워두면 여기저기 사용가능

자바스크립트 소스코드 작성 위치에 따른 종류

  1. inline : 요소 속성에 간단한 자바스크립트 코드를 작성
  2. internal : HTML문서 내에 script태그에 코드를 작성
  3. external : 코드를 별도의 .js파일로 작성해서 읽어오는 방법
    (모듈로 만들어서 export하고 쓰는 쪽에서 import해서 불러오는 방법도 있음)

1. inline 방식

        여는 태그의 이벤트 속성에 간단한 소스코드를 작성하는 방식
        작성할 코드가 매우 소량이거나
        다른 방식과 혼합할 경우 고려해볼 수 있음

2. internal 방식

        HTML문서에 script라는 태그 영역을 만든 뒤
        함수단위로 소스코드를 작성하고
        특정 요소에 이벤트가 발생했을 때 해당 함수를 호출하는 방식
